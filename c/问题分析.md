2. 明确小问划分
本题共 5 个核心小问（外加 Memo），属于层层递进结构。
Task 1: 粉丝投票分布重建 (The Reconstruction)
直接目标： 估算历史上每一周、每一位选手的粉丝投票份额（或排名）。
隐含目标：
解决“多解性”： 证明有多种投票组合会导致相同的结果，因此必须输出“分布”而非“单点值”。
数据清洗： 处理退赛、数据缺失或特殊的双淘汰周。
输出要求： 概率分布图（如小提琴图）、置信区间、确定性指标（如方差或熵）。
Task 2: 争议案例的反事实分析 (The "What If")
直接目标： 针对 Jerry Rice 和 Bobby Bones，互换评分规则（Rank vs. %），判断他们是否会更早被淘汰。
隐含目标：
赛制敏感性分析： 论证哪种赛制更容易保护“低分高人气”选手。
跨赛制数据对齐： 将 Rank 数据转化为 % 数据（或反之）以进行跨规则模拟。
输出要求： 模拟结果对比表（原结果 vs. 新规则下的预测结果）、生存周期变化图。
Task 3: 评委拯救机制的影响 (The Judges' Save)
直接目标： 量化该机制在多大程度上改变了“原本应淘汰者”的命运。
隐含目标：
公平性评估： 评委是否倾向于拯救特定类型的选手？
统计显著性： 这种改变是随机的还是系统性的偏差？
输出要求： 拯救成功率统计、被拯救选手的后续表现追踪。
Task 4: 驱动因素与相关性分析 (The Correlations)
直接目标： 识别哪些特征（性别、职业、舞伴、出场顺序等）与“裁判分”和“粉丝投票”强相关。
隐含目标：
分离变量： 区分哪些因素影响的是“技术分（Judge）”，哪些影响的是“人气分（Fan）”（利用 Task 1 的结果）。
特征重要性排序： 使用机器学习模型（如随机森林）输出 Feature Importance。
输出要求： 相关系数矩阵热力图、回归分析报告。
Task 5: 优化评分系统 (New Scoring System)
直接目标： 提出一套新的评分/淘汰规则。
隐含目标：
多目标优化： 在“技术公平性（裁判权重）”与“娱乐参与度（粉丝权重）”之间寻找平衡点。
验证有效性： 用历史数据验证新规则不会产生极端不合理的淘汰。
输出要求： 新规则的数学公式、模拟对比图、优势论证。

4. 题型定位与方法推荐
结合 2026 MCM Problem C 的大数据特性（Season 1-33 数据量较大），本题属于 「反向仿真与统计推断类」 综合题。

Q1
参数估计 / 逆问题求解
已知 $Y$ 和 $X_1$，求 $X_2$。且 $Y$ 是 $X_1, X_2$ 的非线性/离散组合。
多模型对比（必须）：

1. 蒙特卡洛模拟 (MC): 暴力随机生成投票，筛选符合淘汰结果的样本（最推荐，易于解释分布）。

2. 约束规划 (CP/IP): 求可行解的边界。
Q2
离散事件仿真 (DES)
改变系统规则 $f(\cdot)$，观察输出 $Y$ 的变化。
单模型分析： 复用 Q1 的 MC 模拟器，主要工作是编写不同赛制的逻辑代码。
Q3
统计假设检验
对比两组样本（有Save vs 无Save）的差异。
统计分析： 生存分析 (Survival Analysis/Kaplan-Meier) 或 简单的概率比较。
Q4
关联挖掘 / 回归分析
寻找变量间的相关性。
多模型验证：

1. 线性/Logistic回归： 解释性强，看系数正负。

2. 随机森林/XGBoost： 看 Feature Importance，捕捉非线性关系。
Q5
多目标优化 / 决策理论
设计新函数 $f_{new}$ 以优化某些指标（如公平性、相关性）。
启发式设计： 基于 Q2/Q4 的结论构造公式，利用 Pareto 最优思想平衡裁判与粉丝话语权。

特别提示：
Problem C 的难点通常不在于数学公式的复杂度，而在于 “数据的解释力” 和 “逻辑的严密性”。
在 Q1 中，必须强调你生成的是 “可能的投票分布（Probabilistic Distribution）”，而不是绝对的真值。
在 Q4 中，需区分 相关性（Correlation） 不等于 因果性（Causation）。
